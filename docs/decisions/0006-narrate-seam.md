# 0006 — `POST /narrate` seam: FE host loop ↔ Gemini proxy

## Status
Accepted (prototype phase). Extends ADR 0003 (boundaries + invariants) and ADR
0004 (the §6 seam); gates M3 BUILD. Append-only. Does **not** change the §6 types
in `frontend/src/contracts/` — M3 moves the *source* of `HostDirective.utterance`,
not its shape.

## Context
M3 lights up the **first path that calls the LLM/backend**. M2 left a clean seam:
`frontend/src/lib/host-loop.ts` populates `HostDirective.utterance` from a canned
placeholder (currently `` `listen — eyes on ${who}` ``, line 36). M3 replaces that
*source* with a real, persona-voiced line generated by Gemini, behind a thin
Fastify proxy (`POST /narrate`) so the API key stays server-side (ADR 0003 #6).

Two boundary facts make the seam non-obvious and worth recording:

1. **The proxy is a new HTTP contract adjacent to §6, but it is not a §6 type.**
   §6 types live in `frontend/src/contracts/` and are consumed by the UI. The
   `/narrate` request/response is a backend-validated wire contract between the FE
   host loop and the proxy. Per ADR 0003 the backend **does not import the §6 types
   at runtime** — no cross-workspace coupling — so the request shape must be
   restated as a backend-local schema.
2. **The proxy must receive only an event's SAFE fields.** Brief §10 names the LLM
   inputs as `{type, narrative, confidenceTier, streamer, eventScore}`. Passing the
   whole `PerceptionEvent` would leak perception detail (`vantages`, `embedUrl`,
   `source.ref`, `heatDelta`/`novelty`/`legibility`) across the boundary for no
   narration benefit, eroding contracts-as-seam (ADR 0003 #3) and widening the
   spoiler surface.

## Decision
Define `/narrate` as a minimal request/response contract; the FE calls it behind a
small injectable client interface; the backend validates with its own local zod
schema.

- **Request body — the §10 safe-input projection of a `PerceptionEvent`, nothing
  more:**
  ```ts
  // The shape the FE sends and the backend's local zod schema validates.
  // NOT the whole PerceptionEvent: no vantages, no embedUrl, no source, no heat/novelty/legibility.
  interface NarrateRequest {
    type: PerceptionEvent['type'];       // 'clutch' | 'reveal' | 'drama' | 'launch' | 'irl' | 'other'
    narrative: string;                   // one-line "what's happening" (non-empty)
    confidenceTier: 1 | 2 | 3 | 4;       // drives tier-hedging
    streamer?: string;                   // credit the source/original; optional (matches Vantage.streamer)
    eventScore: number;                  // 0..1, the ranked salience
  }
  ```
- **Response body — one short line, nothing else:**
  ```ts
  interface NarrateResponse {
    utterance: string; // single non-empty line: no newline chars, length-bounded (§10 "max ~20 words")
  }
  ```
  The FE assigns `NarrateResponse.utterance` to `HostDirective.utterance`; the rest
  of the M2 directive flow is unchanged (`speak`/character/cut all consume the same
  `HostDirective` stream off the bus; the cut still renders
  `cutToVantage.embedUrl` verbatim).
- **Backend-local schema, no cross-workspace import.** The zod schema lives in
  `backend/src/modules/narrate/` (module convention mirrors
  `modules/health/`). It restates `NarrateRequest` independently; it does **not**
  `import` from `frontend/src/contracts`. The two stay in sync by review against
  §10, not by a runtime dependency. (If a second backend consumer ever needs the
  §6 types, promote them to a shared package then — ADR 0003 — not now.)
- **FE client behind an interface.** The host loop calls `/narrate` through a small
  injectable client (a one-method async interface returning `{ utterance }`,
  mirroring the `Speaker` seam in `lib/speak.ts`), so tests inject a stub and the
  real `fetch` swaps in without touching the character/player. The base URL comes
  from `VITE_API_BASE_URL` (already wired for staging — ADR 0005), defaulting to the
  local backend in dev.
- **`GEMINI_MODEL` is an env knob; exact model id confirmed at impl time.** The
  proxy reads the model id from `process.env.GEMINI_MODEL` (`.env.example`:
  `gemini-3.1-flash`); the exact Gemini 3.1 Flash Live id/endpoint is verified
  against Google's docs during BUILD (ADR 0003 note). Never hardcoded.

### How the seam structurally backs the M3 invariants
The contract is shaped so each invariant is hard to violate, then *proven* by the
implementer's tests (this ADR records the structure, not the tests):

- **Cost-gating (ADR 0003 #4).** The host loop already only reaches the
  utterance-building branch when an event clears the surface threshold and the
  silence budget (`host-loop.ts`); the `/narrate` call replaces the canned line in
  exactly that branch — so it fires *on a surfacing event only*, never on idle,
  never on a poll/timer. The backend complements this: a malformed body is rejected
  by zod with **`400` before the Gemini client is constructed or called** — the
  proxy refuses to spend on a bad request.
- **Secrets-from-env (ADR 0003 #6).** `GEMINI_API_KEY` is read from
  `process.env` only. It is **not** a request field — `NarrateRequest` has no key
  field, so any `apiKey`/`GEMINI_API_KEY` in the body is unknown to the schema and
  ignored (zod strips/rejects unknown keys); the key never appears in the response
  (`NarrateResponse` is `{ utterance }` only) and is never logged. Keeping the key
  off the wire contract makes leaking it structurally awkward.
- **Spoiler-safety on a generated line (ADR 0003 #1).** The narration is now
  *generated*, so spoiler-safety is enforced **at the prompt**, not by a runtime
  strip in the proxy. `buildNarratePrompt` (`narrate.prompt.ts`) always carries the
  §10 no-spoiler rule and matches register to `confidenceTier` (tier 1 plain; tiers
  2–4 hedged; tier 4 also explicitly unconfirmed); this is the control, and it is
  what's proven by `backend/tests/narrate-prompt.test.ts`. The proxy itself does
  **not** detect or strip an outcome from the model's reply: it returns the model
  text **verbatim** apart from collapsing newlines and trimming to a single line
  (`narrate.routes.ts`). A reliable runtime outcome-strip is infeasible — the proxy
  has no outcome oracle for an arbitrary event, so it cannot know which words in a
  generated line would constitute a spoiler. The complementary FE-side guard stays
  compiler-enforced: every `HostDirective` is still `spoilerSafe: true` (ADR 0004),
  and on a `/narrate` failure the host **degrades to silent** rather than speaking a
  broken/empty line. **Residual risk (accepted prototype posture):** despite the
  prompt, the model could in principle still emit a spoiler; the prompt is the
  control and `spoilerSafe: true` remains the compiler-enforced directive-level
  contract. A stronger runtime guard would require a reliable outcome oracle, which
  does not exist here. Exact tier-hedging copy is tunable per the navigator's §13
  call and is not part of the seam.
- **Official embeds only (ADR 0003 #5).** `/narrate` neither receives nor returns
  any embed/vantage data — the request projection excludes `vantages`/`embedUrl`
  entirely — so narration cannot touch the cut. The player still renders
  `cutToVantage.embedUrl` verbatim from the unchanged directive (M1/M2).

## Consequences
- The implementer has a fixed target: build the backend-local zod schema against
  `NarrateRequest` and return `NarrateResponse`; wire the FE host-loop client
  against the same two shapes. The seam is the only coordination point between the
  backend and FE work — backend contract first, then the FE swap (cross-layer
  order, ADR 0003).
- The request projection (`{type, narrative, confidenceTier, streamer, eventScore}`)
  is the *complete* perception detail crossing the boundary. Anything the prompt
  later wants beyond §10 is a deliberate seam change (new ADR), not an ad-hoc field.
- `frontend/src/contracts/` is untouched; `HostDirective` keeps its M2 shape. The
  directive flow does not reshape — only the utterance's origin moves from canned to
  `/narrate`.
- Backend-local validation means the §6 request projection and the zod schema are
  two restatements of the same §10 list; they drift only if changed without review.
  This is the accepted cost of no cross-workspace runtime coupling (ADR 0003).
- `zod` becomes a backend dependency at M3 (not currently in
  `backend/package.json`); the implementer adds it during GREEN.

## Alternatives considered
- **Send the whole `PerceptionEvent` to `/narrate`.** Simpler call site, but leaks
  `vantages`/`embedUrl`/`source`/heat-novelty-legibility across the boundary,
  widening the spoiler surface and breaking contracts-as-seam for fields the prompt
  never uses. Rejected — the seam should carry only §10's safe inputs.
- **Import the §6 types into the backend for validation** (e.g. a shared types
  package, or a cross-workspace import). Removes the two-restatements drift risk but
  reintroduces cross-workspace runtime coupling that ADR 0003 deliberately avoids
  for the prototype. Rejected now; revisit only if a second backend consumer of §6
  appears.
- **Accept the key (or model id) from the request body** to make the proxy
  configurable per call. Directly violates secrets-from-env and invites a key on the
  wire / in logs. Rejected — key and model come from `process.env` only.
- **Return a structured payload** (e.g. `{ utterance, tier, tokens, model }`).
  Speculative; the UI consumes only the line, and a richer response widens the
  surface that must be proven spoiler-safe and key-free. Rejected — `{ utterance }`
  only, per §6 minimalism (ADR 0004).
- **Call Gemini directly from the browser.** Puts `GEMINI_API_KEY` in client code —
  an immediate secrets-from-env violation and the reason the proxy exists (ADR 0003).
  Rejected.
